# Self-Hosting

## Supported Deployment Modes

### Local

- Single-machine deployment
- Development and testing
- Personal use

### On-Premises

- Organization-managed infrastructure
- Full control over data residency
- Integration with existing security infrastructure

### Air-Gapped

- No network connectivity required
- Suitable for classified or highly regulated environments
- All dependencies bundled

## Requirements

- No external service dependencies for core functionality
- All artifacts available for offline installation
- Configuration via files (no external config services required)

## Operational Considerations

- Logs written locally
- Metrics exportable but not required
- Updates applied manually in air-gapped mode
